I recently re-installed my OS on my three PCs, so I recorded most of the installation scripts, it may help for someone would like to get a new Linux Mint/Ubuntu system for scientific computation.

Some notes:

  • Since Linux Mint is developed based on Ubuntu 18.04, most of the script should work on Ubuntu 18.4 installation as well.
  • It takes about 20 minutes for the Basic installation if a good internet connection is established.
  • If you are in China, better to set the set your repository to some Chinese repositories, such as Ali, wangyi.
  • It is my installation scripts, may not working for everyone (for example, change of versions, url address). Also, some of the installation may need some dependency, I might have just installed them but forgot to mention. So, whenever meet any problem, always learn to Google it.
  • If all essential parts are done, there should be no trouble to compile the code

1 Basic installation

1.1 First initialization (ESSENTIAL)

sudo apt update
sudo apt upgrade

1.2 Install Google Chrome

if you prefer Firefox, then you do not have to install 

sudo su
echo "deb [arch=amd64] http://dl.google.com/linux/chrome/deb/ stable main" > /etc/apt/sources.list.d/chrome.list
wget -q -O - https://dl-ssl.google.com/linux/linux_signing_key.pub | apt-key add -
apt update 
apt install google-chrome-stable

1.7 Compile hypre (ESSENTIAL)

First determine which compiler you are going to compile. Here only show compile Hypre with default gcc and openmpi, the installation will be located at $(HOME)/hypre/gnu. 

# Home directory
cd
git clone https://github.com/hypre-space/hypre.git
mkdir hypre/gnu
cd hypre/src
./configure
make install
cp -r hypre/lib ../gnu
cp -r hypre/include ../gnu

2 Other applications (optional)

2.1 Install Intel compiler

  • Get the free license use the edu email address and download
  • run installation command.
  • I would customize to disable Intel MPI as it may conflict with the openmpi compile
  • If installed with the windows-linux subsystem, there is no GUI available, you may have to disable more from the installation package to make it go through.
  • You will have to source the compilervar.sh in order to use ifort, icc, icpc as your compiler.

2.2 Install Openmpi with Intel compiler

source /opt/intel/compilers_and_libraries_2019.3.199/linux/bin/compilervars.sh
mkdir MPI
mkdir MPI/openmpi-intel
cd Downloads
wget https://download.open-mpi.org/release/open-mpi/v4.0/openmpi-4.0.1.tar.gz
tar -xzvf openmpi-4.0.1.tar.gz
cd openmpi-4.0.1
./configure --prefix=/home/yeldon/MPI/openmpif90 CC=icc CXX=icpc FC=ifort CFLAGS="-g -O2" FCFLAGS="-g -O2" FFLAGS="-g -O2" --enable-mpi1-compatibility
make
make install

You will need the following scripts to activate the openmpi-intel environment. 

source /opt/intel/compilers_and_libraries_2019.3.199/linux/bin/compilervars.sh
export PATH=/home/yeldon/MPI/openmpi-intel/bin:$PATH
export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/home/yeldon/MPI/openmpi-intel/lib

There are three options to do that

  • Every time when you would like this environment, type the following commands to the command window.
  • Write the scripts to a file (for example, ~/.intel_env) and every time when you need the environment, type source ~/.intel_env.
  • Append the scripts to .bashrc file, so that the command will execute automatically when log in.

To see the difference, you may try to type mpif90 --version in command window, you will see something like GCC 7.3.0 if you have not yet activate the environment. If you have activate the environment, you will see something like ifort.

2.4 Install AmrVis

For AmReX code visualization 

sudo apt install libmotif-dev
sudo apt install libxpm-dev
git clone https://github.com/AMReX-Codes/Amrvis
git clone https://ccse.lbl.gov/pub/Downloads/volpack.git
cd volpack
make
cd ..
cd Amrvis
make DIM=1
make DIM=2
make DIM=3
alias amrvis1d=~/Amrvis/amrvis1d.gnu.ex
alias amrvis2d=~/Amrvis/amrvis2d.gnu.ex
alias amrvis3d=~/Amrvis/amrvis3d.gnu.ex
